# lumora.fi

Sources for https://lumora.fi, a static Lumora portfolio site for boudoir and fine art nude photography in Helsinki. The public site is TFP-only: it presents portfolio albums, the collaboration process, contact by email, and draft/reference documents.

The site is generated with [Zola](https://www.getzola.org). It stays static: no backend, no client framework, and no form service in v1.

## Content model

Portfolio source photos live in ignored album folders:

```text
gallery/<slug>/*.jpg
gallery/<slug>/album.toml
```

Each `album.toml` contains the public album metadata: `title`, `description`, `order`, `cover`, optional `cover_position`, and `[[images]]` entries with `file`, `alt`, `featured`, and optional `position`. Zola templates read those sidecars with `load_data()`.

Generated public gallery files are written to `static/gallery/<slug>/`: full WebP files for album pages, `previews/` WebP files for album grids, selective `covers/` WebP files for `album.cover`, and selective `featured/` WebP files for images with `featured = true`. Generated route stubs are written to `content/portfolio/`. The homepage file `content/_index.md` is hand-authored and is not regenerated by the Makefile.

Crop control is template-driven, not Makefile-driven. Use CSS-compatible values such as `cover_position = "48% 28%"` or `position = "50% 35%"` in `album.toml`; templates pass these to `object-position`.

## Build flow

- `make documents` regenerates downloadable PDF documents from `documents/*.md`.
- `make build` regenerates gallery assets, portfolio route stubs, downloadable documents, and runs `zola build`.
- `make build-fast` regenerates downloadable documents and runs `zola build` without regenerating images or content stubs.
- `make minify` runs the production build, minifies CSS, and subsets the local Jost and Gilda Display fonts.
- `make deploy` and `make preview` publish through Cloudflare Pages and should only be run after an explicit release decision.

## Containerized build

The normal build can run without installing Zola, ImageMagick, ExifTool,
Ghostscript, Node tooling, or fonttools on the host. The Makefile uses
Apple's `container` CLI and builds the image for `linux/amd64` with Rosetta
enabled at runtime:

```sh
make docker-build
```

Useful container targets:

- `make docker-build` - rebuild generated gallery assets, regenerate downloadable documents, and run `zola build`.
- `make docker-build-fast` - regenerate downloadable documents and run `zola build` without regenerating images.
- `make docker-documents` - regenerate downloadable PDF documents inside the project container.
- `make docker-minify` - run the full minified production build.
- `make docker-preview` - deploy a Cloudflare Pages preview.
- `make docker-deploy` - deploy production to Cloudflare Pages.

Preview and deploy targets still require an explicit release decision and a
`CLOUDFLARE_API_TOKEN` in the host environment. The token is passed into the
container at runtime and is not baked into the Docker image.

## Documents
